# Handoff — журнал передачи проекта Author

Этот файл — память проекта между сессиями. Правила его ведения — в `CLAUDE.md`.

## В РАБОТЕ

- **Задача:** Свести дизайн-прототип и рабочее приложение. Вернуть в Next-app
  дизайн из docs/design/stol-proba-4.html и выверенный словарь; завести
  ЕДИНЫЕ источники правды, чтобы больше не расходилось. Название проекта —
  «Стол и Кабинет». «диагноз» → «Разбор». Линзы: Не высушить / Сверить /
  Усилить. Порядок правок из очереди — потом.
- **План:**
  1. Достать полный CSS/разметку из прототипа (ветка
     claude/localhost-access-issue-awg3aq, docs/).
  2. Создать источники правды: docs/СЛОВАРЬ.md (каждое слово UI) и
     блок дизайн-токенов; правило в CLAUDE.md «текст — из словаря, цвет — из
     токенов».
  3. Дизайн-токены → app/globals.css (беж #f1e8d2, чернила #2b2416, бирюза
     #0e6f68, карамель #b98a2e, шрифты Georgia/Times/Courier, чердак-полоса,
     ховеры навигации, стили кнопок).
  4. Словарь по коду: бренд «Стол и Кабинет», вводный текст, «диагноз»→
     «Разбор» (контракт + парсер + читатели + legacy-фолбэк), три линзы,
     статусы (Черновик/У наставника/Завершено), кнопки (Депеша наставнику,
     Положить в тетрадь, Письмо мастеру), Голос/Наставники в Кабинете.
  5. Тесты, typecheck, build, скриншоты вживую; PR → main.
- **Критерий готовности:** рабочее приложение выглядит и говорит как прототип;
  есть единый Словарь и токены + правило в CLAUDE.md; «диагноз» нигде не
  осталось; всё зелёное.
