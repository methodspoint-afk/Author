# Handoff — журнал передачи проекта Author

Этот файл — память проекта между сессиями. Правила его ведения — в `CLAUDE.md`.

## В РАБОТЕ

- **Задача:** «Путь фрагмента» (главный пэйофф, ТЗ §5.2/§13.5): увидеть весь
  пройденный путь текста — дифф первой ↔ последней версии, сводка пути в
  словах, лента вех (версии с датами и пометками «что изменилось»).
- **План:**
  1. `lib/path.ts`: чистые `pathSummary` (версий, слов было → стало) и
     `pathMilestones` (дата, пометка, объём, дельта слов по версиям) + тесты.
  2. FragmentPane: раскрывающийся блок «Путь фрагмента» (только при 2+
     версиях) — сводка, полный дифф первая ↔ последняя (diffWords с
     фолбэком), лента вех. В VersionView добавить createdAt; заодно дата
     в тултипе чипов версий (закрывает пункт «даты у чипов»).
  3. Стили в тихом тоне — без графиков и конфетти.
  4. Тесты, typecheck, build, проверка вживую; затем по процедуре
     «main всегда актуален» — PR, слияние, перезапуск ветки.
- **Файлы:** lib/path.ts, components/FragmentPane.tsx,
  app/desk/[notebookId]/page.tsx, app/globals.css, tests/path.test.ts.
- **Критерий готовности:** в тетради с 2+ версиями раскрывается «Путь
  фрагмента»: видно, с чего текст начинался, чем стал, дифф целиком и вехи
  с датами; тетрадь с одной версией блока не показывает. Всё зелёное.
